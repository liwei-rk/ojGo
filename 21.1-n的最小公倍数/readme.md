## 1-n的最小公倍数
```
题目描述
为什么1小时有60分钟，而不是100分钟呢？这是历史上的习惯导致，但也并非纯粹的偶然：
60是个优秀的数字，它的因子比较多。事实上，它是1至6的每个数字的倍数。即1,2,3,4,5,6都是可以除尽60。
我们希望寻找到能除尽1至n的的每个数字的最小整数。

解答要求
时间限制：6000ms, 内存限制：64MB
输入
输入一个整数n,(1<n<=100），测试包含组样例，读到文件末尾结束。

输出
输出1至n的最小公倍数。

样例
输入样例 1

6
10
100
输出样例 1

60
2520
69720375229712477164533808935312303556800
```
```
这个数字可能非常大，比如n=100, 则该数为：
69720375229712477164533808935312303556800
须考虑大数处理，可以用big包
big包里已经有两个数最大公约数计算方法GCD，这里可以利用GCD求两个数的最小公倍数LCM （lcm(a, b) == a * b / gcd(a, b)）
可以逆序遍历1-n，不断求r = lcm(n, n-1), r = max(lcm(r, n-2), r) ...
```
